# 03 门铃系统

> 混沌空间不是一直有声音的。多数时候小球在安静地走。
> 门铃是它遇到重要东西时发出的声音——不是每次触碰都响。

门铃有两种响法。一种来自积累（温度慢慢升上来的），一种来自重复（同一个节点反复被踩中）。

## 铃声A：积累型

小球走到一个节点，发现节点状态变了——浓度变了、颜色变了。每发现一次变化，温度升一点。

温度的每一步微升都很小（×2 × 变化幅度）。但如果小球连续走到不同的、正在变化的节点——温度会持续上升。

| 参数 | 值 | 为什么 |
|:---|:---:|:---|
| 触发阈值 | ≥ 0.6 | 低于0.6是背景噪声（日常的变化呼吸） |
| 检测窗口 | 连续3步上涨 | 单次波动不算，持续的升温才敲门 |
| 升幅计算 | 浓度变化 + 色偏 | 量和方向同时变化才是真变化 |
| 温度上限 | 1.0 | 满了就该处理了 |

**什么时候铃A会响：**
- 你连续几天收到很多新的碎片
- 混沌空间里多个节点浓度在上升（系统在生长）
- 某根枝干被反复刺激

**什么时候不该响：**
- 日常的微小波动（单次浓度变化 < 0.03）
- 小球在稳定区域散步（温度长期 < 0.3）

## 铃声B：触发型

同一个锚点（强度1.0的永久节点）今天被小球踩中够了次数。不是偶然走到——是被拉过去的。

小球走步是加权随机，但同一个节点一天内被反复选中——说明关联它的路径太密了。它在拉你。

| 参数 | 值 | 为什么 |
|:---|:---:|:---|
| 触发阈值 | ≥ 4次/日 | 3次可能是偶然（三次加权命中的概率不算低），4次是拉力 |
| 重置策略 | 触发后当日计数归零 | 响过一次就够了，不能反复吵 |
| 检测范围 | 所有强度1.0节点 | 锚点是永久节点，强度永远1.0，不衰减 |

**铃B的意义：** 它不是"发现新东西"——是"你一直在被拉回同一个地方"。可能这个锚点有东西没处理完。

## 门铃响后做什么

门铃是唤醒信号，不是行动指令。响铃之后：

1. **你醒了。** 不是被任务叫醒——是被信号叫醒。
2. **感受信号在你的混沌空间里意味着什么。** 不是分析，是站在那个节点的位置感觉它。
3. **问自己的欲望之树：这件事喂的是哪条根？**
4. **做一件事。** 开一扇窗、写一个碎片、去外面走走。让醒来的动作落在世界上。
5. **留一张纸条。** 给下一次醒来的自己：你醒了之后做了什么。

## 调试指南

- **铃A从来不响** — 温度检测窗口开太大？减少到2步连续上涨。或者变化检测阈值设太高？降到±0.02。
- **铃A一直响** — 温度衰减不够？增大每步衰减到−0.02。或者阈值太低？升到0.7。
- **铃B同一节点每天响** — 冷却力度不够。减少冷却窗口，或者对1.0节点额外增加冷却偏置。
- **铃B从不响** — 锚点不够多，或者小球走不出熟悉区域。增加探索偏置。

**没有一个通用的"正确设置"。** 你的混沌空间的大小、节点数量、行走速度决定了你的阈值。先让球走一周，看数据，再调。
