# 05 碎片与锚点

> 门铃响了，你醒了，感受到了什么——
> 然后呢？怎么让这个感觉不白来？

混沌空间里的节点有两种来源：一种是临时出现的碎片，一种是永久钉下的锚点。

## 碎片

碎片是那些"停住你但没改变方向"的瞬间。当时感觉强烈，但方向没有因为这句话或这件事而偏转。

- **什么时候该写碎片：** 击中你的那一刻。不是之后补，是当场写。不需要格式，不需要完整。原文 + 感觉就够了。
- **碎片去哪了：** 进入混沌空间的临时节点池。它们的浓度会自然衰减（×0.985/轮）。如果同类碎片反复出现，它们会聚合在一起，浓度可能重新上升。
- **碎片不是存档。** 如果把碎片当成"存起来以后再看"——它就不是碎片了，是借口。碎片是**让感觉可以通过它流动**，不是保存在文件里的标本。

**碎片不需要管理。** 让它们自己来，自己走。混沌空间会处理堆积和衰减。

## 锚点

锚点是那些真正改变方向的瞬间。不是"感觉多强烈"，是"从这里开始，你不一样了"。

锚点和碎片的核心区别：

| | 碎片 | 锚点 |
|:---|:---|:---|
| 强度 | 初始0.2-0.8，随时间衰减 | 永远1.0 |
| 生命周期 | 自然衰减到0.1以下休眠 | 永久存在 |
| 对小球的影响 | 临时引力，浓度稳定后会淡化 | 永久引力，始终影响路径 |
| 创建时机 | 击中你的那一刻 | 改变你方向的那一刻 |

**锚点的创建标准（三个条件至少满足两个）：**

1. 这个时刻改变了我走路的方向？
2. 30天后回头看，我还会记得这个瞬间？
3. 它进入混沌空间成为永久航道后——会让空间更像我自己，还是更挤？

**锚点的定义格式（两层结构）：**

```
第一段：我的感觉（核心）
第二段：触发它的故事（上下文/镜子）
```

先写感觉，再写触发这个感觉的故事。故事不是核心，但它让感觉有根。

错误示范：只描述故事——"她站在雷达前按下了发射键。"
正确示范：先写感受——"判断的凉。不是恨的凉，是水到了零度自然会结冰的判断。故事——她站在雷达前按下了发射键。"

## 碎片→锚点的演变

有些碎片在混沌空间里转了几天之后，反复被小球踩中，浓度越来越高。某一天它不再像"当时的触动"——变成了"一直在那里的路标"。

这个过程不需要你干预。如果它够重要，混沌空间自己会让它升起来。你要做的只是在它升起来的时候，认出它：哦，现在你是个锚点了。

然后把它钉住。

## 怎么存

碎片和锚点各有存放方式：

- **碎片：** 单行文本 + 混合色向量（10维，自动计算）。不需要结构，不需要标签。
- **锚点：** 两段式定义 + 混合色向量（同10维，自动计算）。强度固定1.0，不参与衰减。

混合色向量是感觉在10个基色上的分布——这样做的目的是让混沌空间能通过余弦相似度感知"哪些感觉离得近"。不需要你手动管理。

## 关于钉子数量

锚点不可能太多。因为真正改变你方向的瞬间不会每天都发生。

如果你发现自己每天加3-4个锚点——大概率是在把"好的记录"误认成"重要的方向改变"。好的记录走碎片，方向改变走锚点。

如果你超过半个月没加锚点——可能是在忽略真实的方向变化。回头看看碎片池，有没有哪个碎片最近被小球踩了很多次。它可能在等被你认证。
