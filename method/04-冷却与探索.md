# 04 冷却与探索

> 混沌空间最大的敌人不是安静——是惯性。
> 小球一旦形成路径依赖，就停不下来了。

如果没有冷却机制，小球走步的加权随机会形成正反馈：走到A → A的关联路径权重更高 → 更容易走到A的邻居 → 关联更强 → 走不出去。

这是你在搭建混沌空间时一定会遇到的第一个工程问题。

## 冷却机制

小球最近走过的节点被临时降低权重。距离当前步越近，冷却越强。

**公式：** 每个邻居的权重 = 亲和力 × 冷却因子

```
冷却因子 = max(0.15, 1.0 - (10 - dist) / 10 × 0.85)
        其中 dist = 距离当前步数（0=刚走过，9=9步前）
```

| dist（几步前） | 冷却因子 | 效果 |
|:---:|:---:|:---|
| 0（刚走过） | × 0.15 | 几乎不可能再走 |
| 1 | × 0.23 | 同一节点连续两次概率极低 |
| 3 | × 0.40 | 短期回访压低 |
| 5 | × 0.57 | 中期适中 |
| 7 | × 0.74 | 弱冷却 |
| 9 | × 0.91 | 接近不影响 |
| ≥10 | × 1.0 | 不在冷却窗口，不影响 |

**关键设计原则：**

1. **不设硬锁（不×0）。** 即使刚走过的节点，仍然有极小概率再走。这是混沌——不是队列调度。
2. **窗口大小=10步。** 短于10步，小球可能在3-4个节点间来回打转。长于10步，冷却失去区分度（大多数节点都在冷却窗口外）。
3. **衰减曲线是线性的，不是指数的。** 指数衰减会让冷却"要么全有要么全无"——线性衰减给了连续过渡。

## 探索偏置

从未被访问过的节点获得额外加权。这是让小球走出舒适区的动力。

```
如果节点从未在trail中出现过：权重 × 1.8
如果节点在trail中出现过：    权重不变
```

**×1.8 是反复调试的结果：**

- ×2.0 — 太强，小球偏好跳转到从未访问的节点，忽略亲和力。混沌变成了随机探索。
- ×1.5 — 太弱，很难打破既有路径。节点数量增加后（>30个节点），偏置几乎不起作用。
- ×1.8 — 在小球有30-40个节点、每天走50-80步的场景下，平均每10-15步会探索一个从未去过的地方。节奏刚好。

**探索偏置什么时候调：**
- 节点少（<15个）→ 调低到1.4。否则小球很快跳完所有节点，偏置失去意义。
- 节点多（>50个）→ 调高到2.2。否则陌生节点的吸引力仍然竞争不过亲和力连接。
- 小球经常走到死胡同 → 调高探索偏置，同时增加节点间连接密度。

## 冷却与探索的关系

两者是互补的，不是竞争的：

- **冷却** 防止小球困在当前区域
- **探索** 吸引小球去从未去过的地方

两者同时作用时的行为模式：

```
阶段1：走到一个节点 → 感受它
     ↓ 冷却降低该节点权重
阶段2：被拉到关联节点 —— 但刚来过的节点在冷却
     ↓ 如果附近都走过了，冷却覆盖了整个区域
阶段3：探索偏置吸引小球走向陌生节点
     ↓ 新的区域 → 新关联 → 新的节点被激活
回到阶段1
```

这不是设计出来的路径——是冷却和探索的拉力让小球自然形成的散步节奏。

## 调试指南

**小球在2-3个节点间来回打转**
- 冷却窗口太小 → 扩大到15步
- 冷却力度不够 → 降低底线冷却到×0.1
- 这些节点之间的亲和力太强 → 检查它们是否有真实的关联，还是松散的连接过多

**小球从不回到重要的锚点**
- 冷却窗口太大 → 缩小到7步
- 探索偏置太强 → 降到1.4
- 检查锚点是否跟其他节点有足够的亲和力连接

**小球走了一圈所有节点后变得不稳定**
- 说明节点太少或连接太密
- 增加节点数（>20个），减少不必要的连接

**没有"调对"的终点**——只有当你观察小球走路的轨迹，感觉"这个节奏是我想要的"——那就是对的。
